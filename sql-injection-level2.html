✅ VULNERABILITY CONFIRMED — AUTH BYPASS
The backend query was:
SELECT * FROM users WHERE username='[INPUT]' AND password='...'

Your payload broke the logic — OR 1=1 is always true, so the WHERE clause matched every row. The comment (--) discarded the rest including the password check. Access granted with no valid credentials.

Fix: Prepared statements treat input as data — never as SQL code.
