Part 1 — Reconnaissance & OSINT
Master the first phase of every cyberattack — mapping the target's digital footprint before a single exploit is fired.
🔭 What is Reconnaissance?
Reconnaissance (Recon) is the first and most critical phase of any cyberattack or penetration test. Before exploiting anything, attackers map the target's entire digital footprint — domains, IP addresses, open ports, employee names, technologies in use, and potential vulnerabilities. The quality of recon directly determines the success of everything that follows. All intelligence gathered here powers precision exploits, social engineering, and phishing campaigns.

Passive vs. Active Reconnaissance
🛡 Passive Recon
Uses publicly available sources only — no direct contact with the target system ever occurs
Zero trace — the target has no way to detect or log this activity
Tools: WHOIS, Google Dorking, Shodan, social media scraping, DNS lookups
Examples: reading job postings, WHOIS queries, LinkedIn reconnaissance, Shodan searches
Detection risk: ZERO — fully undetectable
⚡ Active Recon
Directly interacts with the target — sends packets, probes ports, queries DNS servers directly
Leaves logs and network traces — IDS/IPS systems can detect and alert on this
Tools: Nmap port scanner, banner grabbing, ping sweeps, zone transfer requests
Examples: Nmap scan, traceroute, direct DNS zone transfer attempt against target's servers
Detection risk: HIGH — requires written authorization
WHOIS Lookups
📋 What is WHOIS?
WHOIS is a public query protocol that exposes who registered a domain. A single WHOIS lookup can reveal the registrant's full name, organization, email address, phone number, creation and expiration dates, registrar name, and nameservers. This is pure passive recon — all data is pulled from public registrar databases. Attackers use this data to identify key personnel for targeted phishing and to begin mapping DNS infrastructure.
$ whois mega-corp.com
Domain Name: MEGA-CORP.COM
Registrant Name: Marcus J. Holloway
Registrant Organization: MegaCorp Industries
Registrant Email: admin@mega-corp.com
Registrar: GoDaddy LLC
Creation Date: 2009-03-14T00:00:00Z
Name Server: NS1.MEGA-CORP.COM
⚠ Registrant email → spear-phishing target. Nameserver → DNS zone transfer attempt vector.
Google Dorking (Google Hacking)
🔎 What is Google Dorking?
Google Dorking uses advanced search operators to surgically find sensitive information that companies accidentally expose to search engines — config files, password lists, employee directories, internal PDFs, and admin panels. It is entirely passive — you never touch the target server. The Google Hacking Database (GHDB), maintained since 2002, catalogs thousands of known working dork strings used in real penetration tests and attacks.

Shodan — The Hacker's Search Engine
🛰 What is Shodan?
Shodan is a continuously running internet scanner that crawls every publicly accessible IP and records open ports, service banners, software versions, SSL certificate data, and more. Unlike Google — which indexes web content — Shodan indexes devices and services: webcams, routers, industrial SCADA systems, exposed databases, IoT devices, and misconfigured servers. Shodan also automatically flags known CVEs based on detected software versions. A single query can reveal an unpatched target in seconds.
Service Banners
Shodan captures banner responses — including software name and version. Seeing Apache/2.4.1 in a banner instantly reveals if a known CVE applies to that server.
Open Port Discovery
Find every open port on a target without sending packets yourself. Search hostname:mega-corp.com to see everything Shodan has already indexed.
CVE Detection
Shodan cross-references detected software versions against known CVE databases. Vulnerable services appear directly in search results with their CVE numbers listed.
IoT & SCADA
Exposed webcams, industrial control systems, and smart devices with default credentials are all indexed. Many industrial systems still respond to factory-default passwords.
Search Filters
Narrow results with powerful filters: org:"MegaCorp", port:3306, country:US, os:"Windows Server", before:2025-01-01.
Defensive Use ✓
Defenders search their own org name to audit their own exposure — discovering misconfigured services before attackers do. Run it against yourself first.
DNS Enumeration
🌐 What is DNS Enumeration?
DNS (Domain Name System) is the internet's phone book — it maps hostnames to IP addresses and reveals infrastructure details. DNS enumeration extracts all available DNS records for a target to build a complete map of its servers, mail systems, subdomains, and hosting providers. Tools like nslookup, dig, dnsrecon, and dnsenum automate this. Passive DNS enumeration queries public databases without ever touching the target's own nameservers.

Key OSINT Tools
Maltego
Visualizes relationships between entities — people, domains, IPs, social profiles, and organizations. Pre-installed in Kali Linux. Maps org structures graphically.
Recon-ng
A modular Python recon framework that ingests multiple OSINT modules and consolidates findings into one unified report. Pre-installed in Kali Linux.
theHarvester
Passively harvests emails, subdomains, IPs, and hostnames from Google, Bing, Shodan, LinkedIn, and more. Core passive recon tool in Kali Linux.
Sublist3r
Python tool for fast subdomain enumeration by scraping search engines and certificate transparency logs. Finds hidden infrastructure like dev and staging servers.
Wayback Machine
Archives historical web snapshots. Attackers find old admin portals, config files, and past technologies that may expose legacy vulnerabilities still running internally.
SpiderFoot
Automated OSINT scanner querying over 1,000 public sources. Runs from Kali CLI or a graphical web interface. Correlates findings across all sources automatically.
REAL WORLD2013 Target Breach — 40M Cards Stolen via OSINT-Driven Supply Chain Attack
Attackers used passive OSINT recon to identify a third-party HVAC vendor with network access to Target's systems. The breach resulted in 40 million credit card numbers and 70 million customer records stolen — all starting with public information gathering.
1
OSINT Phase: LinkedIn, WHOIS records, and public procurement contracts revealed Fazio Mechanical — a Target HVAC contractor with remote network access credentials.
2
Spear Phishing: Using harvested employee names, roles, and email formats, a targeted phishing email compromised Fazio's login credentials.
3
Lateral Movement: Attacker used Fazio's VPN credentials to pivot from the vendor's network into Target's internal payment card infrastructure.
4
Deployment: POS malware (BlackPOS) was quietly deployed across 1,800+ Target retail locations, silently capturing card data at every swipe.
5
Result: $162M in damages, resignation of Target's CIO and CEO, class-action lawsuits. It all started with a WHOIS lookup and a LinkedIn profile.
